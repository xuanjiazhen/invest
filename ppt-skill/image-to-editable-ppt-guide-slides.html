Part 1 · 安装 — Codex 环境（推荐）

安装 Skill

在 Codex 对话中输入以下任一命令：

/install image-to-editable-ppt

或指定 GitHub 地址安装：

安装 image-to-editable-ppt 这个 skill，
地址是 https://github.com/ningzimu/image-to-editable-ppt-skill

使用 Skill

$image-to-editable-ppt 把这张图片转成可编辑 PPT

支持多种输入方式：

  • 直接粘贴/拖拽图片到对话框
  • 提供本地文件路径
  • 附加 PDF 或图片版 PPTX 文件

Part 1 · 其他平台 + OCR Token 配置

Claude Code / 其他 Agent

  • Claude Code：安装 skill 后自然语言触发
    "帮我把这张截图转成可编辑的 PPT"
  • 其他平台：支持 skill 加载 + 文件读写 + CLI 执行 + page worker 分派的 agent 均可使用

参照各平台的 skill 安装文档

OCR Token（强烈推荐）

  1. 访问 百度 AI Studio 申请 Access Token
  2. 个人免费额度完全够用
  3. AI 首次使用时会询问，发送 Token 即可
  4. 自动写入 ~/.editppt/config.yaml
    Windows: %USERPROFILE%\\.editppt\\config.yaml

⚠️ 不配置也可运行，但文字还原质量有折扣

API Backend 配置（按需）

通常不需要手动配置。仅在 Codex OAuth 不可用且需第三方 API 时，告诉 AI 服务商、base URL、模型名和 key 即可自动配置。密钥存入用户级配置，输出中自动遮蔽。

Part 2 · 输入 → 输出

输入类型输出说明
1 张图片1 页 .pptx单页重建，文本→文本框，几何→形状，视觉→图片资产
多张图片多页 .pptx按提供顺序排列，每图一页
多页 PDF页数一致 .pptxPDF 第 N 页 → 输出第 N 页
图片版 PPTX页数一致 .pptx保留原页面备注（不翻译、不摘要、不改写）

文本 & 形状还原

  • 可读文字 → 原生 PowerPoint 文本框
  • 简单几何 → PowerPoint 形状（矩形/圆形/线条）
  • 文字大小/位置由 OCR 测量数据驱动
  • 同级文字字号自动保持一致

复杂视觉处理

  • 照片、插画、纹理 → 独立图片资产
  • 可移动但内部不可编辑
  • 带来源记录的 asset sheet
  • 表格/图表优先保留可编辑语义

Part 2 · 工作流程

四阶段流水线

阶段动作
① Prepare输入归一化 → 每页 source.png + OCR 文字标注
② Dispatch按 max_concurrent_pages 并行分派 page worker
③ Record每页重建 → 自检 → 修正（可能多轮迭代）
④ Finalize按页序组装 .pptx + deck 校验

输出目录结构

output/image-to-editable-ppt/{job-id}/
├── input/
├── deck_manifest.json
├── page_jobs.json
├── run_state.json
├── final/{name}_edited.pptx
└── pages/page_001/ ... page_00N/

Part 3 · 注意事项 — 模型选择

模型评级说明
GPT-5.5（ChatGPT Pro）推荐流程控制稳定，表现最好。推荐 Pro 用户使用
GPT-5（ChatGPT Plus）谨慎可用但 token 消耗高，10 页 PPT 可能耗尽 5 小时额度
DeepSeek不可用不支持识图。中间步骤（源图+标注→JSON）会直接编造内容。等 DeepSeek 支持识图后再尝试
GPT-5.5 以下不保证模型基础理解能力和 skill 遵循能力不足

Token 消耗警示

  • 转换成本 = 生成图片 PPT 的 2-3 倍
  • 单页复原 > 10 分钟
  • 10 页可能耗尽 Plus 5h 额度
  • AI 会执行"重建→自检→修正"多轮迭代

轻量替代方案

如果只是对某一页 PPT 图片不满意：
直接用 gpt-image-2 的图像编辑能力，发给它针对性修改。
比走完整 skill 流程轻量得多。

从零生成 PPT → 用 codex-ppt-skill

Part 3 · 注意事项 — 能力边界

✅ 能做

  • 图片/PDF/图片版PPTX → 可编辑 .pptx
  • 文字 → 原生文本框（可调字体/大小/颜色）
  • 简单形状 → PowerPoint 原生形状
  • 保留输入 PPTX 的页面备注
  • 多页并行处理

❌ 不能 / 有限

  • 不能 100% 复刻：文字位置可能有轻微偏移
  • 照片/插画/纹理 → 仅作为图片资产移动，内部不可编辑
  • 表格/图表 → 低置信度时保留为资产
  • 不是从零生成 PPT（那是 codex-ppt-skill 的事）

运行前提

  • Agent 支持 page worker/subagent 分派
  • 可用 editppt image backend
  • OCR Token 配置为推荐项
08 / 08

总结

三步上手

  1. 安装：/install image-to-editable-ppt
  2. 配置 OCR：百度 AI Studio 申请 Token → 发给 AI
  3. 使用：$image-to-editable-ppt 把图片转成可编辑 PPT

三条忠告

  1. 没有可编辑需求别用——token 成本高
  2. DeepSeek 不要试——不识图会编造
  3. 不追求 100% 复刻——文字位置可能有偏移

适用场景：已有图片/PDF → 需要二次编辑文字和元素位置。
