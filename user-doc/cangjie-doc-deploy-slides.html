一、Kotlin · 关键差异展开

平台站有的、语言站没有

  • Android 应用开发完整指南：首个 App 创建、Kotlin 常用模式、协程在 Android 中的使用
  • Android Framework API 参考：android.* 包在 Kotlin 视图下的完整文档
  • Kotlin-first 理念阐述：Google I/O 2019 宣布 Kotlin 优先

语言站有的、平台站没有

  • 完整语言参考：语法/类型系统/函数/类/泛型等权威文档
  • 标准库 API 参考：stdlib / kotlin.test / kotlinx.coroutines / kotlinx.serialization 等
  • Kotlin Multiplatform 完整文档：跨平台开发指南
  • 服务端/数据科学：Spring Boot / Ktor / DataFrame 教程

关键事实：developer.android.com/kotlin/learn 页是一份有实质内容的速成教程（含变量声明、类型推断、空安全、条件语句、类与属性等），不是纯链接页。但深入的语言参考通过"官方语言参考"链接指向 kotlinlang.org。

二、Swift · 关键差异展开

平台站 developer.apple.com

  • /swift/：功能展示页（Expressive / Safe / Fast / Adaptable）+ 代码片段
  • /swift/resources/：资源门户，含 Xcode 下载、语言指南外链、API 文档、视频、发行说明、Swift 徽标
  • /documentation/swift/：标准库 API 参考（与 iOS/macOS 框架 API 同站）
  • /tutorials/develop-in-swift/：交互式入门教程

语言站 docs.swift.org / swift.org

  • docs.swift.org：The Swift Programming Language 权威书籍（从 About Swift 到语言参考的完整内容）
  • swift.org：开源项目主页，含源码、Bug 追踪、论坛、开发构建下载
  • swift.org/documentation/：社区文档索引（指向各子项目文档）

关键事实：Apple 将语言学习（完整书籍）与平台开发（API 参考+教程+视频）分站部署。平台站 /swift/ 不是语言指南而是营销+功能展示页——真正的语言学习需跳转至 docs.swift.org。这与开发者直觉可能有差异。

四、仓颉 · 关键发现

language site → platform site 链接

  • cangjie-lang.cn 首页"开发文档"按钮直接跳转至 developer.huawei.com 仓颉专区
  • "入门指南"按钮指向语言站自身 /docs 路径
  • 语言站顶部导航：文档 / 下载 / 开源仓库 / 中心仓

内容分工现状

  • 语言站独有：语言规范/设计、标准库完整文档
  • 平台站独有：Kit 开发指南、实践文章、FAQ、鸿蒙 API 参考
  • 双站重叠：语言学习内容、工具链文档——均以不同形式存在

与 Kotlin 相似之处

  • 独立语言站承载语言本身文档
  • 平台站承载平台集成开发指南
  • 语言基础内容双站均有（速成 vs 完整）

与 Swift 相似之处

  • 同一公司拥有语言和平台
  • 语言站到平台站有直接链接
  • 标准库 API 托管位置尚在讨论中

关键事实：仓颉是调研范围内唯一同时满足"同公司 + 独立语言站 + 双站内容均有实质积累"的语言。Kotlin（跨公司）、Swift（语言站为社区/书籍站）、ArkTS（无独立语言站）均不兼具此特征。
