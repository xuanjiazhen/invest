## 图结构

| 模块 | 布局 | 内容 |
|------|------|------|
| 1. 鸿蒙应用开发 | 左上模块 | 开发代码、自动化测试、线上用户 |
| 2. 故障数据采集 | 左中模块 | 本地端、CI/CD、线上诊断端；采集错误日志、内存快照、GC 日志 |
| 3. 故障定位智能体 | 右上主模块 | 数据解析 Agent、代码分析 Agent、上下文感知、模式推理、证据链组装、修复生成、模式库检索 Agent、修复生成 Agent |
| 4. 故障模式库 | 底部中间模块 | 模式建模、结构化存储、向量存储、版本管理 |
| 5. 来源机制 | 左下模块 | 专家录入、案例反馈、社区贡献、自动挖掘 |
| 6. 自动集成 | 右下模块 | DevEco Studio、CI/CD、线上诊断端到 Coding Copilot、门禁阻断/修复建议、秒级定位/Bug 单 |

### 数据流

- 鸿蒙应用开发 → 故障数据采集 → 故障定位智能体
- 故障定位智能体内部：上下文感知 → 模式推理 → 证据链组装 → 修复生成
- 来源机制 → 故障模式库，标注为“模式沉淀”
- 故障定位智能体 ↔ 故障模式库，标注为“智能体 / 模式库”
- 故障定位智能体 → 自动集成
- 自动集成内部三条链路：IDE → 一键修复，CI/CD → 门禁阻断/修复建议，线上诊断端 → 秒级定位/Bug 单
