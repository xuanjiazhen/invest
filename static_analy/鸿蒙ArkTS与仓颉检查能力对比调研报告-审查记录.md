## 4. PPT 转化检查清单

PPT 制作时需要满足以下约束：

1. 不使用 HTML 作为最终产物。
2. 标题应尽量是结论句，而不是单纯主题词。
3. 每页只放一个主证据对象：矩阵、双栏对比、分层图、差异拆解图、流程图或表格。
4. 封面、章节、结束页继承模板视觉。
5. 页脚保留正式汇报材料的页码/保密提示语法。
6. 总览矩阵必须包含“能力项、大类、ArkTS、ArkTS 工具、仓颉、仓颉工具、差异结论”。
7. HiChecker 页必须标注“开发期运行检测，不属于编译期静态检查”。
8. 来源与边界页必须保留仓颉华为授权页不可读说明。

## 5. 剩余限制

1. 仓颉华为开发者联盟授权文档未能访问，未确认是否存在 DevEco Studio 内部集成或 HarmonyOS 应用框架补充规则。
2. 仓颉公开 cjlint 文档面向命令行静态检查，不覆盖所有可能由 IDE 或编译器提供的体验。
3. ArkTS 规则数量较多，本报告按能力族总结，不逐条解释所有规则。
4. PPT 将以汇报可读性为优先，不把完整 Markdown 表格逐字搬入每一页。
