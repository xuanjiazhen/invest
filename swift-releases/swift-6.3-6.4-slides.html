🔗 @c 属性 — Swift ↔ C

用 @c 将 Swift 函数暴露给 C 代码，编译器自动生成头文件声明。

Swift// 暴露给 C @c func callFromC() { print("Hello!") }
C header (生成)void callFromC(void);
Swift// 自定义 C 名称 @c(MyLib_callFromC) func callFromC() { }
C header (生成)void MyLib_callFromC(void);
Swift// 实现 C 头中声明的函数（Swift 校验签名） @c @implementation func callFromC() { print("Swift impl") } @c @implementation func add(_ a: Int, _ b: Int) -> Int { a + b }

6.3 适用于函数和枚举。C → Swift 方向亦可用。

🔒 所有权系统与非可复制类型

~Copyable — 所有权唯一，防止意外复制

Swiftstruct Buffer: ~Copyable { let ptr: UnsafeMutablePointer<Int> consuming func dispose() { ptr.deallocate() } } struct FileHandle: ~Copyable { let fd: Int32 borrowing func read() -> Data { /* 借用读取 */ } mutating func write(_ data: Data) { /* 可变写入 */ } consuming func close() { /* 消费后不可再用 */ } }

6.4 consuming / borrowing / mutating 三种所有权模式，编译期保证无 use-after-free。
